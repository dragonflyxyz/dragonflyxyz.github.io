（2）The Model Object

正如Sketchup模块代表整个Sketchup的应用程序，Model Object代表一个独立的SketchUp文件（*.skp），或更准确的说，设计信息包含在这个文件中。当在SketchUp中打开一个新文件时，Sketchup模块保持原样，但当前的Model object的数据完全不同。
Model class 中的methods提供关于当前设计的信息，例如，

title 这个method得到当前设计的标题
description 这个method得到设计的文本描述
path 这个method得到当前设计的文件路径

（3）The Entities Object

每个SketchUp设计中的几何对象都可以以一个Entity或它的子类表示，包括线、面、图像、文本、组等等组件。为管理或修改Entity对象，你需要访问设计的主要Entities容器，这个容器有以下3个作用：

  • 添加新的Entity对象
  • 移动、缩放、选择、删除Entity对象
  • 在可通过索引访问的数组中存储Entity对象

其中第一条最重要，Entities类包括很多方法添加新的Entity对象，如下：

  • add_line - creates an Edge object from two points
  • add_edges - forms an array of Edge objects from a series of points
  • add_circle - forms an array of Edge objects that combine to form a circle
  • add_ngon - forms an array of Edge objects that combine to form a polygon
  • add_face - creates a Face object from edges or points
  • add_text - adds a label to the design at a given point

3.3 Edges

Edge对象通过Entities类的add_line方法创建。如：

大多数Edge类的方法都可划分到以下两个种类中的其中一个中：
（1）Edge外观的设置方法
（2）连接到Edge的对象方法

第一种类的方法是设置Edge的可见属性的。在Drawingelement类提供的hidden方法中，Edge提供了soft和smooth方法。最重要的是分辨隐藏线和平滑线的区别：平滑的曲线结合相邻表面成一个单一的表面(通常是弯曲的), 而隐藏线不改变相邻表面。
第二种类中，all_connected方法返回包含连接到Edge的所有Entity对象的数组。相类似的是faces方法返回包含连接到Edge的所有Face对象的数组。

在SketchUp中，Edge的端点用Vertex对象表示。Edge类包含一些与Vertex交互的方法：

  • vertices ：返回Edge的两个Vertex对象的数组
  • start / end ： 返回Edge的起始/终止Vertex对象
  • other_vertex ：给定Edge的一个Vertex对象，返回另一个
  • used_by? ： 判断一个Vertex是否连接到Edge

在Edge类中有两个方法不属于任何一个种类：length和split。
length 返回Edge对应的线段长度
split 在Edge对象的线上用一个点建立第二个Edge对象。当调用split后，第一个Edge对象只能延续到给定的点，而第二个Edge对象则从给定的点到原来线的终点。

Ruby Scripts

重要的三点：

  • SketchUp脚本必须用Ruby语言编写；
  • 不加密的脚本文件后缀为.rb，加密脚本文件后缀为 .rbs；

    load “star.rb”

    也可将star.rb放到子目录中，如放置到插件目录的ch3子目录下，可如下方式调用：
    load “ch3/star.rb”

    也可放到其他目录中，调用绝对路径
    load “c:/ruby/ch3/star.rb”

平移（Translation）

例如，想移动一个形状从[1, 2, 3]到[3, 2, 1]，转换需要使用数组[2, 0, -2]，这个三元素数组定义了方向和距离，通常被称为平移向量。
SketchUp API提供3种方法来创建转换：

  • 调用Geom::Transformation.new来使用平移向量
  • 调用Geom::Transformation.translation来使用平移向量

  • 自身使用三元素平移向量

转换坐标系统（Transforming Coordinate Systems）

当你创建一个有4个轮胎的汽车模型，旋转汽车时，整个车体围绕车的起源点旋转。但当你旋转一个轮胎时，旋转则相关与轮胎的轴。出于转换的目的，每个轮胎都有其自身的起源点和轴。这种分离的坐标系统称为局部坐标系统，整个车的坐标系统称为全局坐标系统。

将一种坐标系统转换为另一种坐标系统的几种方法：

  • 调用Geom:Transformation.new（使用新起源点和Z轴）
  • 调用Geom:Transformation.new（使用新起源点和X轴、Y轴）
  • 调用Geom:Transformation.new（使用新的X轴、Y轴、Z轴和起源点）
  • 调用Geom:Transformation.axes（使用新起源点，X轴、Y轴、Z轴）
