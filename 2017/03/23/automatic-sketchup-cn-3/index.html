第11章 SketchUp接口：视图、相机、页、工具

前2个对象：视图（View）和相机（Camera）定义了在SketchUp窗口中的表现。
页（Pages）和工具（Tools）可以通过插件建立和控制更多的图形元素。

11.1 视图（View）

像素信息（Pixel Information）

视图类涉及到像素和设计窗口的6个方法：

  • vpheight 返回设计窗口的高度（单位：像素）
  • vpwidth 返回设计窗口的宽度（单位：像素）
  • corner 返回窗口一个角的像素坐标
  • center 返回窗口中心的像素坐标
  • screen_coords 返回一个三维点的像素坐标

  • pixels_to_model 转换一个点的像素尺寸到设计尺寸

11.5 工具类（The Tool Class）

在这里，Ruby并没有通常的接口。但类似的Tool类并不意味着被扩展。 工具类确定的方法是供SketchUp查找一个tool是活动的且用户执行一个动作。大多数Tool类的方法可以归纳到以下两类：
(1) Life-cycle 关联到Tool的活动和操作的方法
(2) Event-handling - 在响应用户事件中调用的方法

SketchUp工具的生命周期（The Life Cycle of a SketchUp Tool）

在任何时间，在SketchUp工具条上只有一个工具能被激活。默认情况下，激活工具是选择工具。为在代码中激活工具，需要调用Model类的select_tool方法。例如如想使my_tool激活，需要执行下列命令：

当一个新工具被激活，SketchUp调用它的activate方法。这在这个工具执行事件前要执行初始化任务。当工具完成其操作变为非激活时，deactivate方法执行处理回收程序。

工具事件执行（Tool Event Handling ）

当工具的用户接口设计好并激活，SketchUp调用工具的方法来响应。例如当my_tool是激活的，且用户在窗口中双击鼠标，SketchUp找到my_tool对象，试图调用它的onLButtonDoubleClick方法。my_tool的onLButtonDoubleClick中并没有代码，但只要有，SketchUp就会在双击鼠标时调用这个方法。

onMouseEnter view 当鼠标进入设计窗口时调用

InputPoints将二维坐标转换为三维坐标，其中最重要的两个方法是：

  • pick 提供了InputPonit的像素坐标
  • position 返回设计中点的位置

pick有两种调用方法：
(1) pick x, y, view - 在view中将input_point和二维位置联合在一起

在pick和position使用中，有用的InputPoint方法有：edge, face, vertex 。当一个InputPoint位于一个实体上，这些方法返回附着的Edge，Face和Vertex。这在一个工具需要修改几何体时很有用。

set_status_text方法用来设置VCB中的文本。它有2个参数：String和一个常量（这个常量定义文本是如何显示的）
这个常量有3种值：
(1) SB_PROMPT - 在左下角紧贴几个图标的位置显示（不在VCB中）
(2) SB_VCB_LABEL - 在VCB左侧显示
(3) SB_VCB_VALUE - 在VCB内显示
